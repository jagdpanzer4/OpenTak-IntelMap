Zero zmian w bazowym UI · instalacja przez Plugin Manager · niezależne aktualizacje

✅ Rekomendowane podejście
Python Backend Plugin + React SPA w iframe
Jeden plik .whl instalowany przez Plugin Manager. Backend Flask serwuje zbudowaną aplikację React. OpenTAK UI ładuje ją w iframe pod zakładką "UI". Żadnych modyfikacji w bazowym OpenTAK UI.
Struktura repo (ots-maptak)
ots-maptak/ ├── ots_maptak/ # Python package │ ├── __init__.py │ ├── app.py # Flask Blueprint │ ├── default_config.py │ └── ui/ # zbudowany Vite │ ├── index.html │ └── assets/ │ ├── index.js │ └── index.css │ ├── maptak-ui/ # React + Vite źródła │ ├── src/ │ │ ├── MapTAK.tsx │ │ ├── components/ │ │ ├── map/ │ │ └── hooks/ │ ├── package.json │ └── vite.config.ts │ └── pyproject.toml # Poetry build
Jak to działa w OpenTAK
# 1. Zainstaluj plugin (.whl) Plugin Manager → Upload Plugin → restart OpenTAKServer # 2. Dostęp do mapy OpenTAK UI → Plugins → MapTAK → zakładka "UI" → iframe z pełną mapą # 3. Ten sam origin socket.io ✓ # ciasteczka auth /api/map_state ✓ /api/missions ✓ # Żadnych CORS, żadnych tokenów # 4. Update pluginu Plugin Manager → Delete → Upload # Bazowy UI NIEZMIENIONY
