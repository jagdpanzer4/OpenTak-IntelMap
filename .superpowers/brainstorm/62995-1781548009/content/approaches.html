3 podejścia do naprawy mapy OpenTAK

React 19 + TypeScript + Leaflet 1.9 już zainstalowane w OpenTAK UI 1.7.5

A

Napraw istniejący komponent mapy

Rozszerz bieżący MapView w OpenTAK UI — dodaj WebSocket CoT, warstwy kafelków, ślady EUD, strefy. Minimalna ingerencja w codebase.

Zalety

  • Leaflet już zainstalowany
  • Łatwy PR / merge upstream
  • Mniejsze ryzyko regresji

Wady

  • Ograniczony przez obecną architekturę
  • Brak WebGL / wektorowych kafelków
  • Leaflet 1.x starzeje się

Czas: 2–3 dni   Ryzyko: Niskie

B

🔥 Nowy komponent MapTAK (plugin) — Leaflet + pełny CoT stack

Zupełnie nowy komponent MapTAK jako oddzielny plugin/moduł React. Zastępuje stary widok mapy. Pełna integracja z OpenTAK API (WebSocket, REST) i CoT schema.

Zalety

  • Czysta architektura, w pełni testowalna
  • Leaflet + react-leaflet (już w repo)
  • Pełne: ślady, warstwy, strefy, ikony MIL
  • Offline tiles (MBTiles/XYZ local)
  • Łatwy swap w przyszłości na MapLibre

Wady

  • Tygodniowy nakład pracy
  • Wymaga dobrej znajomości CoT API

Czas: 5–10 dni   Ryzyko: Średnie

C

Migracja do MapLibre GL + własna warstwa CoT

Wymiana Leaflet → MapLibre GL JS (WebGL, wektorowe kafelki). Nowy plugin z pełnym stackiem: CoT WebSocket, klastry, 3D budynki, offline PMTiles.

Zalety

  • Najlepsza wydajność (WebGL)
  • Wektorowe kafelki (offline PMTiles)
  • Skalowalność na tysiące jednostek
  • Profesjonalny wygląd (styl OpenMapTiles)

Wady

  • Wymaga wymiany całej warstwy mapowej
  • react-map-gl lub maplibre-react nowa zależność
  • 2–3 tygodnie solidnej roboty

Czas: 15–20 dni   Ryzyko: Wysokie

💡 Rekomendacja: Opcja B — korzysta z już zainstalowanego Leaflet, dostarcza pełne funkcje taktyczne i ma czystą architekturę gotową na przyszłą migrację do MapLibre.
